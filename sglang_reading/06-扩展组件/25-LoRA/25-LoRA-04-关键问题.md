# LoRA：关键问题

## Q1：max_loras_per_batch 与 GPU 内存？

**Explain：** 每槽位占用 `num_layers × rank × hidden × 2`（A+B）量级显存；增大 batch 内 adapter 数线性增显存。应结合 eviction 与业务并发 uid 数调参。

**Code：**

```python
# 来源：python/sglang/srt/lora/lora_manager.py L78-L78
        self.max_loras_per_batch: int = max_loras_per_batch
```

**Comment：** OOM 时优先降 `max_loras_per_batch` 或 rank，而非盲目加 GPU。

---

## Q2：请求未 load 的 adapter 会怎样？

**Explain：** Scheduler 在 batch 构造前调用 load；失败则 abort 或回退 base（视配置）。应用层应预 load 热点 adapter。

**Code（正确 — 预注册）：**

```bash
python -m sglang.launch_server \
 --model-path base-model \
 --enable-lora \
 --max-loras-per-batch 8 \
 --lora-paths skill_a=/path/a skill_b=/path/b
```

**Code（运行时 load API）：**

```python
# 来源：python/sglang/srt/lora/lora_manager.py L46
from sglang.srt.managers.io_struct import LoRAUpdateOutput
```

**Comment：** `lora_strict_loading=True` 时权重 shape 不匹配直接 fail fast。

## Q3：target_modules 不匹配

**Explain：** Adapter 的 `target_modules` 与 `--lora-target-modules` 交集决定实际挂层；`auto_detect_lora_target_modules` 可扫描 base 模型。

**Code：**

```python
# 来源：python/sglang/srt/lora/lora.py L176-L188
        layer_id = get_layer_id(name)
        if layer_id is not None:
            self.layers[layer_id].weights[name] = loaded_weight.cpu()
        elif "embed_tokens" in name or "lm_head" in name:
            # Check if this module is declared in target_modules before loading.
            # When normalized_target_modules is {"all"} (e.g. target_modules was
            # "all-linear"), we allow loading since the server-level
            # --lora-target-modules will govern which modules are active.
            module_name = "embed_tokens" if "embed_tokens" in name else "lm_head"
            if (
                "all" in normalized_target_modules
                or module_name in normalized_target_modules
            ):
```

**Comment：** embedding/lm_head LoRA 需显式允许；否则 silently skip。

## Q4：Eviction 误伤活跃 adapter？

**Explain：** `select_victim` 只从 **非当前 batch 必需** 的 candidates 中选；drainer 防止 unload in-flight 槽位。

**Code（反模式 — 手动 evict 不检查引用）：**

```python
# 错误：直接清空 pool 槽位而不 drainer
mem_pool.slots[uid] = EMPTY_SLOT # 可能 forward 仍在使用
```

**Code（框架路径）：**

```python
# 来源：python/sglang/srt/lora/eviction_policy.py L64-L71
    def select_victim(self, candidates: Set[Optional[str]]) -> Optional[str]:
        """Select the least recently used adapter from candidates."""
        # Iterate through access_order (oldest first) to find LRU victim
        for uid in list(self.access_order.keys()):
            if uid in candidates:
                logger.debug(f"Selected LoRA {uid} for eviction (LRU)")
                self.eviction_count += 1
                return uid
```

**Comment：** candidates 由 Manager 计算，排除当前 batch 活跃 uid。

## Q6：added_tokens LoRA

**Explain：** Adapter 扩展词表时 `lora_added_tokens_size > 0`，embedding 层分配 extra buffer；token id ≥ base_vocab_size 走 extra embedding 路径。

**Code：**

```python
# 来源：python/sglang/srt/lora/lora_config.py L58-L60
        self.lora_added_tokens_size = (
            len(self.added_tokens_config) if self.added_tokens_config is not None else 0
        )
```

**Comment：** 客户端 tokenizer 必须同步 added_tokens.json，否则 id 映射错误。
