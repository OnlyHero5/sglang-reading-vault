## 端到端执行链（阶段 III 验收图）

```mermaid
sequenceDiagram
 participant SCH as 07 Scheduler
 participant TW as TpWorker
 participant MR as 11 ModelRunner
 participant MD as 13/14 Model
 participant ATTN as 17 Attention<br/>(阶段 IV)

 SCH->>TW: ScheduleBatch (extend/decode)
 Note over TW: model_runner.forward(batch)
 TW->>MR: ForwardBatch + ForwardMode
 MR->>MD: model.forward(input_ids, positions, ...)
 MD->>ATTN: RadixAttention / MoE 层
 ATTN-->>MD: hidden states
 MD-->>MR: logits
 MR-->>TW: next_token_logits
 TW-->>SCH: GenerationBatchResult
```

**Explain：** `TpWorker` 持有 `ModelRunner` 实例；Scheduler 子进程通过 ZMQ/共享内存把 batch 交给 worker。`ModelRunner` 负责把 `ScheduleBatch` 转为 `ForwardBatch`、选择 CUDA Graph replay 或 eager forward，并调用 `model.forward`。权重在启动时由 `12 ModelLoader` 灌入，模型类由 `13/14 Registry` 解析。

## 验证建议（零基础可试）

1. **启动日志：** 观察 `Load weight end` 与 `Capture cuda graph` 顺序，确认 12→11 初始化完成。
2. **Registry：** 换 `--model-path` 为 Qwen3，日志应出现 `qwen3` entry class 而非 llama。
3. **Graph：** `--disable-cuda-graph` 对比吞吐；decode 密集场景 graph 通常更优。
