# 专题统计

## 如何使用

统计用于发现内容失衡，不按固定文档套件计数。一个主题可以根据读者任务拥有不同数量的文档。

## 按框架和主题

```dataview
TABLE WITHOUT ID
  key AS "框架 / 主题",
  length(rows) AS "文档数"
FROM "sglang_reading" OR "slime_reading" OR "flash-attn_reading" OR "AI-Infra课程"
WHERE framework
GROUP BY framework + " / " + topic
SORT length(rows) DESC
```

## 按学习角色

```dataview
TABLE WITHOUT ID
  key AS "学习角色",
  length(rows) AS "文档数"
FROM "sglang_reading" OR "slime_reading" OR "flash-attn_reading" OR "AI-Infra课程"
WHERE learning_role
GROUP BY learning_role
SORT length(rows) DESC
```

## 最近更新

```dataview
TABLE framework AS "框架", topic AS "主题", type AS "类型", updated AS "更新"
FROM "sglang_reading" OR "slime_reading" OR "flash-attn_reading" OR "AI-Infra课程"
SORT updated DESC
LIMIT 30
```

## 判断标准

- 核心课程必须保持短而连续。
- Reference 可以深，但必须由入口页分发。
- Troubleshooting 必须包含症状、源码入口、操作和预期。
- Practice 必须能执行或明确说明环境限制。

统计只能提示“值得审阅”，不能证明缺失或质量。文档少可能因为主题聚合得当，文档多也可能只是重复；最终判断仍需入口主线、读者任务和源码证据。
