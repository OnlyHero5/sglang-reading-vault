# 文档类型地图

## 类型职责

| type | 读者问题 |
|------|----------|
| guide | 我应该怎么学、先读什么 |
| map | 模块在哪里、与谁相连 |
| concept | 我需要建立什么心理模型 |
| walkthrough | 一个真实对象如何穿过源码 |
| dataflow | 对象跨进程、rank、GPU 或语言时如何变化 |
| troubleshooting | 症状如何定位到配置、指标和源码 |
| exercise | 如何通过实验证明自己真的理解 |
| reference | 术语、文件地图、版本和补充信息 |
| dashboard | 哪些动态内容满足当前筛选条件 |
| template | 新文档应遵守什么结构与证据占位 |

`type` 描述读者任务，不代表质量等级。短 map 可以是核心入口，长 walkthrough 也可能只适合按需查阅；学习顺序由 `learning_role` 和入口页共同决定。

## 动态分布

```dataview
TABLE WITHOUT ID
  key AS "类型",
  length(rows) AS "篇数"
FROM "sglang_reading" OR "slime_reading" OR "flash-attn_reading" OR "AI-Infra课程"
WHERE type
GROUP BY type
SORT length(rows) DESC
```

## 图谱过滤

```text
[type:guide] OR [type:map]
```

用于查看入口骨架。学习主线可使用：

```text
[learning_role:core]
```

排障内容：

```text
[type:troubleshooting]
```

实践内容：

```text
[learning_role:practice]
```

验证方式：随机选择一篇 `walkthrough`、`troubleshooting` 和 `exercise`。预期：分别能看到对象生命周期、症状→操作→预期、可执行验收；若正文任务与 type 不符，应修正文档而不是只改查询。
