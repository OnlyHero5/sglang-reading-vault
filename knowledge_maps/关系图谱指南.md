# 关系图谱指南

## 如何使用

图谱适合回答“这篇笔记和哪些主题相连”，不适合代替课程目录。首次学习从 [[AI-Infra入门课程]] 开始；进入具体专题后，再用 Backlinks 和 Local Graph 查看上下游。

图中的边只证明文档存在 Wikilink，不自动证明源码调用、运行时依赖或性能因果。要判断真实调用仍需源码/trace，要判断性能仍需固定 workload 的实验。

## 推荐过滤

全库学习内容：

```text
(path:AI-Infra课程 OR path:sglang_reading OR path:slime_reading OR path:flash-attn_reading OR path:knowledge_maps) -path:模板
```

只看核心学习内容：

```text
[learning_role:core] (path:AI-Infra课程 OR path:sglang_reading OR path:slime_reading OR path:flash-attn_reading)
```

只看源码走读：

```text
[type:walkthrough] (path:sglang_reading OR path:slime_reading OR path:flash-attn_reading)
```

只看排障与实验：

```text
([type:troubleshooting] OR [type:exercise]) (path:AI-Infra课程 OR path:sglang_reading OR path:slime_reading OR path:flash-attn_reading)
```

不要使用裸 `-path:sglang` 或 `-path:flash-attn`，它们可能同时排除名称相近的阅读目录。

Obsidian 版本或插件若不支持某段搜索语法，先在 Search 面板逐项组合 `path:` 与 Property 条件；不要为了图谱效果改动文档 Properties。

## 颜色分组

颜色只表达少量稳定维度，避免把图谱变成彩色目录树。

| 查询 | 建议颜色 | 含义 |
|------|----------|------|
| `[framework:cross-framework]` | 金 | 课程与跨框架地图 |
| `[framework:sglang]` | 青 | 推理 serving |
| `[framework:slime]` | 绿 | RL 后训练 |
| `[framework:flash-attn]` | 橙 | Attention 算子 |
| `[type:troubleshooting]` | 红 | 故障定位 |
| `[type:exercise]` | 灰 | 实验与学习检查 |

## Local Graph 起点

| 问题 | 起点 |
|------|------|
| 从 prompt 追到新权重 | [[从Prompt到新权重]] |
| 看 HTTP 请求生命周期 | [[SGLang-HTTP请求全链路]] |
| 看调度与缓存关系 | [[SGLang-Scheduler]] · [[SGLang-RadixAttention]] |
| 看 RL 样本闭环 | [[Slime-RL训练全链路]] |
| 看 Attention IO 与 kernel | [[FlashAttention-前向全链路]] · [[FlashAttention-Attention-IO]] |

Local Graph 建议深度设为 1；只有在追跨库关系时再升到 2。节点过多时先按 `framework`、`type` 或 `learning_role` 过滤。

验证方式：任选一个起点，把图中两条边分别回到正文链接语境。预期：能说清“前置、下游、对照、排障或参考”中的哪种关系；若只能说“它们相连”，这条图谱信息还不足以指导学习。

## 结合其他 Obsidian 功能

- 用 Bookmarks 固定课程首页、当前主线和正在做的实验。
- 用 Backlinks 判断一篇笔记被哪些专题依赖。
- 用 Properties/Bases 做筛选和清单，不依赖文件名排序。
- 用 Search 查源码对象、配置项和错误文本；用图谱看语义邻居。
