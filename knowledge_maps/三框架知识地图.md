# 三框架知识地图

## 如何使用

本页按系统问题对照三库。横向关系表示概念和接口边界，不表示 SGLang 实际部署一定选择 FlashAttention；真实 backend 必须通过配置、日志或 profiler 确认。

## 架构边界

| 系统问题 | Slime | SGLang | FlashAttention |
|----------|-------|--------|----------------|
| 主循环 | generate → train → update | request → batch → forward → response | QK → online softmax → PV |
| 核心状态 | Sample、ObjectRef、updater/engine version | Req、ScheduleBatch、KV location | params、tile、row max/sum、LSE |
| 并发边界 | Ray Actor、Megatron rank | coroutine、进程、ZMQ、GPU rank | CTA、warp、stream |
| 主要压力 | 样本吞吐、训练稳定、同步一致性 | TTFT、TPOT、KV 容量、排队 | HBM traffic、occupancy、编译组合 |

## 一条联合主线

[[从Prompt到新权重]] 是三库共同入口。对象关系表示同一闭环中的不同作用域，不是 id 一一相等：

| Slime | SGLang | Kernel |
|-------|--------|--------|
| driver `rollout_id` / Sample aggregation id | request `rid` | 无业务 request id，只接 tensor/metadata |
| Sample tokens | Req input/output ids | Q/K/V 和 KV 地址 |
| rollout logprob | sampling/logprob 输出 | 无直接同名对象；attention output 只是 logits 的上游 |
| updater/engine version、Sample `weight_versions` | engine model version、路径相关 cache 失效 | kernel 参数不保存模型版本 |

验证方式：从 [[从Prompt到新权重]] 任选一条 Sample，填写上述三列的“可观测、不可观测、需要 profiler/trace 补充”。预期：不会从时间戳猜 request id，也不会仅凭框架名猜 attention kernel。
