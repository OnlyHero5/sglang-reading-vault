# 三框架知识地图

## 如何使用

本页按系统问题对照三库。横向关系表示概念和接口边界，不表示 SGLang 实际部署一定选择 FlashAttention；真实 backend 必须通过配置、日志或 profiler 确认。

## 架构边界

| 系统问题 | Slime | SGLang | FlashAttention |
|----------|-------|--------|----------------|
| 主循环 | generate → train → update | request → batch → forward → response | QK → online softmax → PV |
| 核心状态 | Sample、ObjectRef、weight version | Req、ScheduleBatch、KV slot | params、tile、row max/sum、LSE |
| 并发边界 | Ray Actor、Megatron rank | coroutine、进程、ZMQ、GPU rank | CTA、warp、stream |
| 主要压力 | 样本吞吐、训练稳定、同步一致性 | TTFT、TPOT、KV 容量、排队 | HBM traffic、occupancy、编译组合 |

## 一条联合主线

[[从Prompt到新权重]] 是三库共同入口。对象对应关系：

| Slime | SGLang | Kernel |
|-------|--------|--------|
| rollout_id | rid | 无业务 id，只接 tensor/metadata |
| Sample tokens | Req input/output ids | Q/K/V 和 KV 地址 |
| rollout logprob | sampling/logprob 输出 | logits 上游的 attention 输出 |
| weight version | engine model version、KV flush | kernel 参数不保存模型版本 |
