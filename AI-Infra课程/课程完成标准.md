# AI Infra 课程完成标准

## 通过标准

完成不等于把所有文档标记为已读。读者需要能独立完成以下任务。

## 系统建模

- [ ] 画出 HTTP 请求、调度 batch、GPU forward、文本回程。
- [ ] 画出 Sample、DP split、Megatron train、weight update。
- [ ] 画出 Q/K/V、参数包、dispatch、tile loop、O/LSE。
- [ ] 用同一张图连接 rollout_id、rid、KV slot 和 weight version。

## 源码定位

- [ ] 从一个外部 API 找到内部对象和下一跳消费者。
- [ ] 找到一个状态变量的创建、修改和释放位置。
- [ ] 用分支条件解释一次配置分叉。
- [ ] 用测试或运行现象验证判断，而不是只引用注释。

## 性能工程

- [ ] 区分 TTFT、TPOT、throughput 和 goodput。
- [ ] 区分 HBM 容量、HBM 带宽和计算利用率。
- [ ] 设计只改变一个变量的对照实验。
- [ ] 报告版本、硬件、workload、配置和重复次数。

## 正确性与排障

- [ ] 解释 rid、rollout_id、weight version 的一致性作用。
- [ ] 给出 collective hang 的检查顺序。
- [ ] 区分 Scheduler、Detokenizer、kernel 和训练 loss 问题。
- [ ] 完成至少一个故障注入并记录恢复结果。

## 最终复述

不看文档，用十分钟复述 [[从Prompt到新权重]]。如果中途只能说“框架会处理”，回到对应系统主线补齐对象、边界和证据。
